## 용도에 따라 a와 button를 분리해서 사용해야 한다?!

✅ 분리하기 위한 체크리스트

1. 페이지의 이동 or 팝업 페이지의 호출을 위해 사용되는가? -> `<a>`
2. 스크립트 작동 등 **기능**을 위해 사용되는가? -> `<button>`
3. 둘 다 아님! 그냥 클릭 이벤트가 있는 영역임을 알려주기 위해 사용된다! -> css - cursor: pointer;

✅ 왜 분리해서 사용해야 하는가?

`<a>`태그 안에 `<button>`태그를 넣으면 웹 접근성이 저하된다.
`<a>`와 `<button>`은 둘 다 상호작용을 위한(이벤트가 발생하는) 태그이기 때문에
일부 페이지에서는 두 개의 이벤트가 충돌하면서 한 가지가 무시되거나 먹통이 되는 현상이 발생할 수 있다. (event flow에 따른 결과값)

```html
// 🔴BAD
<a>
  <button>BAD</button>
</a>

// 🔴BAD : 웹 접근성에 어긋나는 마크업임

<a>
  <h1>BAD</h1>
</a>
```

즉, `<a>`태그 안에는 상호작용이 없는 요소(div, span...)만 가능하다.
